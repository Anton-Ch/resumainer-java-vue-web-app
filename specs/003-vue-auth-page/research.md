# Research: Vue Auth Page

**Date**: 2026-06-02
**Source**: Context7 MCP, PostgreSQL 17 docs, PrimeVue 4 docs, Spring Framework 6.2 docs, Flyway docs, BA Data Dictionary

## 1. UUID Strategy — Hybrid (Built-in gen_random_uuid + BIGSERIAL)

| Aspect | Decision |
|---|---|
| **Entity tables PK** (`users`, `contact_detail`) | PostgreSQL built-in `gen_random_uuid()` (UUID v4). No custom generators, no extensions. |
| **Lookup tables PK** (`role`, `user_status`, `user_permission`, `language`) | `BIGSERIAL` (GENERATED BY DEFAULT AS IDENTITY). 2–6 rows each, never exposed externally. |
| **Why hybrid** | UUID provides enumeration protection for entity PKs that may appear in URLs. BIGSERIAL is more efficient for small lookup tables. Built-in `gen_random_uuid()` requires zero setup — no Java library, no PL/pgSQL function. |

**Alternatives considered:**
- UUID v7 (requires `java-uuid-generator` library or custom PL/pgSQL — extra complexity, rejected per user decision)
- `pg_uuidv7` extension (requires superuser install — rejected as unnecessary complexity)
- Auto-increment everywhere (enumeration vulnerability on entity PKs exposed in URLs)

## 2. PrimeVue 4 Form + Zod Resolver

| Aspect | Decision |
|---|---|
| **Form library** | PrimeVue 4 `Form` component with `zodResolver` from `@primevue/forms/resolvers/zod` |
| **Password visibility** | PrimeVue `Password` with `toggleMask` prop — built-in eye icon |
| **Validation** | Zod schema: `z.string().min(1).email()` for email, `z.string().min(8).regex()` for password |
| **Error display** | PrimeVue `Message` component with `severity="error" size="small" variant="simple"` |

**Best practice pattern:**
```vue
<Form :resolver="zodResolver(schema)" @submit="onSubmit">
  <FormField name="email">
    <InputText type="email" placeholder="Email" fluid />
    <Message v-if="$field?.invalid" severity="error" size="small" variant="simple">
      {{ $field.error?.message }}
    </Message>
  </FormField>
  <Password name="password" placeholder="Password" :feedback="false" toggleMask fluid />
  <Button type="submit" label="Log in" />
</Form>
```

## 3. Spring MVC Auth (no Spring Security)

| Aspect | Decision |
|---|---|
| **Auth mechanism** | Server-side HttpSession with `HandlerInterceptor` registered via `WebMvcConfigurer.addInterceptors()` |
| **Session check** | Interceptor `preHandle()`: check `session.getAttribute("user")`, return 401 redirect if null |
| **Rate limiting** | In-memory `ConcurrentHashMap<String, AttemptRecord>` in AuthService. 5 attempts → 15 min lockout. |
| **Spring docs note** | Interceptors are "not recommended as primary security layer" but appropriate for Capstone scope. |

**Alternative considered:**
- Spring Security: Overkill for Capstone, steep configuration curve, conflicts with non-Boot setup
- JWT: Adds token refresh/expiry complexity, no significant benefit for single-server app

## 4. Flyway Migration Strategy

| Aspect | Decision |
|---|---|
| **Naming** | Standard `V{num}__{description}.sql` (e.g., `V1__create_role_table.sql`) |
| **Location** | `backend/src/main/resources/db/migration/` |
| **Placeholder replacement** | Not used (no env-specific SQL needed) |
| **Migration order** | Lookup tables first → users → contact_detail → seed data |
| **UUID v7 function** | Created as a separate migration before tables that use it |

## 5. Database Schema from BA Docs

**Source**: `ba-docs/docs/04_domain-and-data-model/data_dictionary.md`

Existing entities adapted for UUID v7:
- `role` → UUID PK instead of Integer
- `user_status` → UUID PK instead of Integer
- `user_permission` → UUID PK instead of Integer
- `language` → UUID PK instead of Integer
- `users` → UUID PK instead of Integer AUTO_INCREMENT. All FK columns changed to UUID type.
- `contact_detail` → UUID PK instead of Integer. `user_id` FK → UUID.

**Business rules preserved:**
- Table name `users` (plural, avoids PostgreSQL reserved word)
- Email used for authentication (not username)
- Username collected later in My Profile (not in this feature)
- Deactivation uses `status_id` = BLOCKED, not record deletion
